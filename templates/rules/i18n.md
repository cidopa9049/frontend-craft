# 国际化 (i18n) 规则

凡是涉及用户可见文案、提示信息、表单标签、错误消息或 UI 文本时，都应用本文件。

<!-- 请根据项目实际 i18n 方案修改 -->

## 核心原则

- 所有用户可见文案必须使用 i18n key，禁止硬编码中文或英文字符串
- i18n key 按模块组织，使用点分命名：`module.component.label`
- 相同语义的文案复用同一个 key，避免重复定义

## 文案规范

### 命名约定

```
common.confirm        → 通用确认
common.cancel         → 通用取消
user.login.title      → 用户登录标题
order.list.empty      → 订单列表空状态
```

### 分类

- `common.*` — 跨模块通用文案（确认、取消、提交、加载中等）
- `module.component.*` — 模块级文案
- `validation.*` — 表单校验提示
- `error.*` — 错误信息

## 实现指导

### Vue（vue-i18n）

```vue
<template>
  <!-- 正确 -->
  <span>{{ $t('order.status.pending') }}</span>
  <!-- 错误：硬编码 -->
  <span>待处理</span>
</template>
```

### React（react-intl / react-i18next）

```tsx
// 正确
<FormattedMessage id="order.status.pending" />
// 或
const { t } = useTranslation();
<span>{t('order.status.pending')}</span>

// 错误：硬编码
<span>待处理</span>
```

## 检查清单

- [ ] 所有新增用户可见文案使用 i18n key
- [ ] 新增 key 已添加到所有语言文件
- [ ] key 命名符合模块.组件.用途 的约定
- [ ] 包含变量的文案使用插值而非拼接
- [ ] 复数形式使用 i18n 复数规则而非条件判断
- [ ] 日期、数字、货币使用 i18n 格式化

## 反模式

- 在模板或 JSX 中直接写中文字符串
- 使用字符串拼接代替 i18n 插值（`"共" + count + "条"` → `t('common.total', { count })`）
- 在 JS/TS 逻辑中硬编码提示信息
- 不同模块为相同含义的文案创建不同的 key
